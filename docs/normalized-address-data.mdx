## Address Normalization

Datafiniti normalizes all street addresses in our property database.  We convert common street designations to 2-3 character abbreviations.  These conversions are shown below.

### Property Search API Example

Let's say you want to search for the following address:

- 3407 Pennsylvania Avenue,  Austin, TX 78721

In this case, we will need to abbreviate **suffix street name** `Avenue` to Datafiniti's normalization. For this example we will use the following property search API query to show how `Avenue` would be abbreviated to `Ave`. Below is what address you would need to use in the API query. For a full list of possible values Datafiniti normalizes, see [Possible Values for Property Fields](/docs/possible-values-for-property-fields).

```json
{
  "query":"address:\"3407 Pennsylvania Ave\" AND city:Austin AND province:TX AND postalCode:78721",
  "num_records":1
}
```

### Normalized Terms for Street Addresses Code

For a more detailed overview on how Datafiniti's approach to normalizing street addresses and how this process enhances the usability and consistency of address-related data. \
\
This Markdown file documents the **Address Abbreviation & Normalization Toolkit** in the Datafiniti GitHub repo (<a href="https://github.com/datafiniti/use-cases/blob/main/Datafiniti%20Integration%20Use%20Cases/Datafiniti%20Normalization/Address_Abbreviator.md" target="_blank" rel="noopener noreferrer">**source**</a>). It provides tools to standardize addresses by abbreviating street suffixes (e.g., "Street" → "ST", "Avenue" → "AVE"), directional terms (e.g., "North" → "N"), and regions, improving data consistency and API query performance (especially for Datafiniti).

#### **Code Implementations**

| **File**                                | **Language** | **Key Traits**                                                                                                                |
| --------------------------------------- | ------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| `AddressAbbreviator.java`               | Java         | Token-based replacement on all matches; preserves punctuation. Ideal for bulk processing.                                     |
| `DF_Address_Abbreviator_integration.py` | Python       | Context-aware (abbreviates only primary suffix, handles "UNIT" logic, directions, title casing). Best for Datafiniti queries. |

#### **Features & Usage**

- **Dictionaries**: Covers US/Canada street suffixes, directions; extensible.

- **Java Example**: `abbreviateAddress("123 Main Street, Alberta, Northwest")` → `"123 Main ST, AB, NW"`.

- **Python Example**: `normalize_address("123 Main Street Unit 5 North")` → `"123 Main St UNIT 5 N"`.

- **Datafiniti Tie-in**: Enhances queries like `address:"123 Main ST" AND postalCode:"12345*"` for better matches and deduplication.
